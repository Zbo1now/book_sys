# 书友平台 数据库表设计文档

> 数据库：MySQL `project`，ORM：JPA/Hibernate，主键策略：`GenerationType.IDENTITY`（自增）

---

## 表一览

| 序号 | 表名 | 用途 | 核心关联 |
|------|------|------|----------|
| 1 | `user_account` | 用户账户 | — |
| 2 | `user_follow` | 用户关注关系 | `follower_id` / `following_id` → `user_account.id` |
| 3 | `book` | 图书信息 | — |
| 4 | `book_review` | 图书评论/评分 | `book_code` → `book.code`, `user_id` → `user_account.id` |
| 5 | `book_review_like` | 评论点赞 | `review_id` → `book_review.id`, `user_id` → `user_account.id` |
| 6 | `booklist` | 书单 | `creator_id` → `user_account.id` |
| 7 | `booklist_comment` | 书单评论 | `booklist_code` → `booklist.code`, `user_id` → `user_account.id` |
| 8 | `booklist_follow` | 书单关注 | `booklist_code` → `booklist.code`, `user_id` → `user_account.id` |
| 9 | `booklist_like` | 书单点赞 | `booklist_code` → `booklist.code`, `user_id` → `user_account.id` |
| 10 | `activity` | 活动/事件 | `organizer_id` → `user_account.id` |
| 11 | `activity_participant` | 活动报名 | `activity_code` → `activity.code`, `user_id` → `user_account.id` |
| 12 | `private_message` | 站内私信 | `sender_id` / `receiver_id` → `user_account.id` |

## 1. `user_account` — 用户账户

核心用户表，存储所有注册用户信息。角色分为 `USER`（普通用户）和 `ADMIN`（管理员）。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 用户ID |
| `username` | VARCHAR(64) | NOT NULL, UNIQUE | 用户名 |
| `email` | VARCHAR(128) | NOT NULL, UNIQUE | 邮箱（登录凭证） |
| `password_hash` | VARCHAR(255) | NOT NULL | BCrypt 密码哈希 |
| `avatar_url` | VARCHAR(255) | NULL | 头像路径（如 `/avatars/u_1_xxx.jpg`） |
| `title` | VARCHAR(64) | NULL | 个人头衔 |
| `bio` | VARCHAR(500) | NULL | 个人简介 |
| `phone` | VARCHAR(32) | NULL | 手机号 |
| `role` | VARCHAR(32) | NOT NULL, DEFAULT 'USER' | 角色：`USER` / `ADMIN` |
| `status` | INT | NOT NULL, DEFAULT 1 | 账户状态（1=正常） |
| `created_at` | DATETIME | NOT NULL | 注册时间 |
| `updated_at` | DATETIME | NOT NULL | 最后更新时间 |

**索引**：`username`(UNIQUE), `email`(UNIQUE)

---

## 2. `user_follow` — 用户关注关系

记录用户之间的关注/粉丝关系。关注关系有唯一约束，防止重复关注。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 记录ID |
| `follower_id` | BIGINT | NOT NULL | 关注者ID → `user_account.id` |
| `following_id` | BIGINT | NOT NULL | 被关注者ID → `user_account.id` |
| `created_at` | DATETIME | NOT NULL | 关注时间 |

**索引**：`UNIQUE(follower_id, following_id)` — 防止重复关注

**业务规则**：
- `follower_id` 关注了 `following_id`
- 关注数 = `COUNT(*) WHERE follower_id = ?`
- 粉丝数 = `COUNT(*) WHERE following_id = ?`

## 3. `book` — 图书信息

图书目录，存储所有可被评论和加入书单的图书。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 内部ID |
| `code` | VARCHAR(64) | NOT NULL, UNIQUE | 业务唯一标识（如 `b-1`） |
| `title` | VARCHAR(128) | NOT NULL | 书名 |
| `author` | VARCHAR(128) | NOT NULL | 作者 |
| `tag` | VARCHAR(32) | NULL | 标签（如"热门"） |
| `rating` | DOUBLE | NOT NULL, DEFAULT 0 | 综合评分 |
| `reviews` | INT | NOT NULL, DEFAULT 0 | 评论总数 |
| `cover_url` | VARCHAR(255) | NULL | 封面图片路径 |
| `description` | VARCHAR(1000) | NULL | 简介 |
| `long_description` | VARCHAR(4000) | NULL | 详细介绍 |
| `author_intro` | VARCHAR(2000) | NULL | 作者简介 |
| `language` | VARCHAR(64) | NULL | 语言 |
| `category` | VARCHAR(128) | NULL | 分类 |
| `publish_date` | VARCHAR(32) | NULL | 出版日期 |
| `publisher` | VARCHAR(128) | NULL | 出版社 |
| `pages` | INT | NULL | 页数 |
| `isbn` | VARCHAR(64) | NULL | ISBN 号 |
| `featured` | BOOLEAN | NOT NULL, DEFAULT FALSE | 是否推荐/精选 |
| `created_at` | DATETIME | NOT NULL | 创建时间 |
| `updated_at` | DATETIME | NOT NULL | 更新时间 |

**索引**：`code`(UNIQUE)

| 字段 | 类型 | 约束 | 说明 |

| `id` | BIGINT | PK, AUTO_INCREMENT | 评论ID |
| `book_code` | VARCHAR(64) | NOT NULL | 图书code → `book.code` |
| `user_id` | BIGINT | NOT NULL | 评论者ID → `user_account.id` |
| `rating` | DOUBLE | NOT NULL | 评分（1-5） |
| `content` | VARCHAR(2000) | NULL | 评论内容 |

| `updated_at` | DATETIME | NULL | 编辑时间 |

## 6. `booklist` — 书单

用户创建的书单，可包含多本图书。支持关注和点赞。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 内部ID |
| `code` | VARCHAR(64) | NOT NULL, UNIQUE | 业务唯一标识（如 `bl-1`） |
| `title` | VARCHAR(128) | NOT NULL | 书单标题 |
| `description` | VARCHAR(1000) | NULL | 描述 |
| `cover_url` | VARCHAR(255) | NULL | 封面图路径 |
| `creator_id` | BIGINT | NOT NULL | 创建者ID → `user_account.id` |
| `book_codes` | VARCHAR(2000) | NULL | 包含的图书code列表（逗号分隔） |
| `book_count` | INT | NOT NULL, DEFAULT 0 | 包含图书数量 |
| `rating` | DOUBLE | NOT NULL, DEFAULT 0 | 综合评分 |
| `is_public` | BOOLEAN | NOT NULL, DEFAULT TRUE | 是否公开 |

**注**：关注数、点赞数由 `booklist_follow` / `booklist_like` 实时 COUNT 得出，不存储在 `booklist` 表中。

## 10. `activity` — 活动

用户发起的线上/线下活动，需管理员审批后方可展示。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 内部ID |
| `code` | VARCHAR(64) | NOT NULL, UNIQUE | 业务唯一标识（如 `a-1`） |
| `title` | VARCHAR(128) | NOT NULL | 活动主题 |
| `description` | VARCHAR(1000) | NULL | 活动内容描述 |
| `cover_url` | VARCHAR(255) | NULL | 封面图 |
| `organizer_id` | BIGINT | NOT NULL | 发起人ID → `user_account.id` |
| `start_date` | VARCHAR(64) | NULL | 开始时间（ISO 字符串） |
| `end_date` | VARCHAR(64) | NULL | 结束时间（ISO 字符串） |
| `location` | VARCHAR(128) | NULL | 活动地点 |
| `participant_count` | INT | NOT NULL, DEFAULT 0 | 当前报名人数 |
| `max_participants` | INT | NULL | 人数上限（NULL=不限） |
| `status` | VARCHAR(32) | NOT NULL, DEFAULT 'upcoming' | 生命周期状态 |
| `approval_status` | VARCHAR(32) | NOT NULL, DEFAULT 'pending' | 审批状态 |
| `activity_type` | VARCHAR(32) | DEFAULT 'offline' | 活动类型：`online` / `offline` |
| `created_at` | DATETIME | NOT NULL | 创建时间 |
| `updated_at` | DATETIME | NOT NULL | 更新时间 |

**索引**：`code`(UNIQUE)

**状态说明**：

| 字段 | 可选值 | 含义 |
|------|--------|------|
| `status` | `upcoming` | 待开始（当前时间 < startDate） |
| | `ongoing` | 进行中（startDate ≤ 当前时间 < endDate） |
| | `ended` | 已结束（当前时间 ≥ endDate） |
| `approval_status` | `pending` | 待审核（用户刚创建） |
| | `approved` | 审核通过（对外可见） |
| | `rejected` | 已拒绝（不可见） |

**状态流转**：
```
用户创建 → pending（待审核）
    ├→ approved → upcoming → ongoing → ended（时间自动驱动）
    └→ rejected（终态）
```

`status` 由系统根据服务器当前时间与 `start_date`/`end_date` 的对比自动计算（每次读取时同步），无需手动推进。

---

## 11. `activity_participant` — 活动报名

记录用户报名参加活动的关系。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 记录ID |
| `activity_code` | VARCHAR(64) | NULL | 活动code → `activity.code` |
| `activity_id` | BIGINT | NULL | 活动ID → `activity.id` |
| `user_id` | BIGINT | NOT NULL | 报名用户ID → `user_account.id` |

**业务逻辑**：
- 报名时 `participant_count + 1`，取消时 `participant_count - 1`
- 如设置 `max_participants > 0`，达到上限后拒绝新报名

---

## 12. `private_message` — 站内私信

用户之间的一对一私信。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | BIGINT | PK, AUTO_INCREMENT | 消息ID |
| `sender_id` | BIGINT | NOT NULL | 发送者ID → `user_account.id` |
| `receiver_id` | BIGINT | NOT NULL | 接收者ID → `user_account.id` |
| `content` | VARCHAR(2000) | NOT NULL | 消息内容 |
| `is_read` | BOOLEAN | NOT NULL, DEFAULT FALSE | 是否已读 |
| `created_at` | DATETIME | NOT NULL | 发送时间 |

**索引**：`idx_pm_sender(sender_id)`, `idx_pm_receiver(receiver_id)`

**业务逻辑**：
- 未读数 = `COUNT(*) WHERE receiver_id = ? AND is_read = FALSE`
- 会话列表按 sender_id 聚合，取最新一条消息
- 点击会话后将该会话所有未读消息标记为已读

## ER 关系概要

```
user_account (1) ──< user_follow >── (1) user_account
                  ├─ follower_id
                  └─ following_id

user_account (1) ──< book_review >── (1) book
user_account (1) ──< book_review_like >── (1) book_review

user_account (1) ──< booklist >──< booklist_comment >── (1) user_account
                  ├─ creator_id       ├─ user_id
                  │                   └─ booklist_code
                  ├─< booklist_follow (user_id)
                  └─< booklist_like (user_id)

user_account (1) ──< activity >──< activity_participant >── (1) user_account
                  └─ organizer_id     └─ user_id

user_account (1) ──< private_message >── (1) user_account
                  ├─ sender_id
                  └─ receiver_id
```

## 完整建表 SQL

```sql
-- 1. user_account
CREATE TABLE user_account (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(64) NOT NULL UNIQUE,
    email VARCHAR(128) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    avatar_url VARCHAR(255),
    title VARCHAR(64),
    bio VARCHAR(500),
    phone VARCHAR(32),
    role VARCHAR(32) NOT NULL DEFAULT 'USER',
    status INT NOT NULL DEFAULT 1,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);

-- 2. user_follow
CREATE TABLE user_follow (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    follower_id BIGINT NOT NULL,
    following_id BIGINT NOT NULL,
    created_at DATETIME NOT NULL,
    UNIQUE KEY uk_user_follow (follower_id, following_id)
);

-- 3. book
CREATE TABLE book (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    code VARCHAR(64) NOT NULL UNIQUE,
    title VARCHAR(128) NOT NULL,
    author VARCHAR(128) NOT NULL,
    tag VARCHAR(32),
    rating DOUBLE NOT NULL DEFAULT 0,
    reviews INT NOT NULL DEFAULT 0,
    cover_url VARCHAR(255),
    description VARCHAR(1000),
    long_description VARCHAR(4000),
    author_intro VARCHAR(2000),
    language VARCHAR(64),
    category VARCHAR(128),
    publish_date VARCHAR(32),
    publisher VARCHAR(128),
    pages INT,
    isbn VARCHAR(64),
    featured BOOLEAN NOT NULL DEFAULT FALSE,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);

-- 4. book_review
CREATE TABLE book_review (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    book_code VARCHAR(64) NOT NULL,
    user_id BIGINT NOT NULL,
    rating DOUBLE NOT NULL,
    content VARCHAR(2000),
    created_at DATETIME NOT NULL,
    updated_at DATETIME,
    INDEX idx_book_review_book (book_code),
    INDEX idx_book_review_user (user_id)
);

-- 5. book_review_like
CREATE TABLE book_review_like (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    review_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    UNIQUE KEY (review_id, user_id)
);

-- 6. booklist
CREATE TABLE booklist (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    code VARCHAR(64) NOT NULL UNIQUE,
    title VARCHAR(128) NOT NULL,
    description VARCHAR(1000),
    cover_url VARCHAR(255),
    creator_id BIGINT NOT NULL,
    book_codes VARCHAR(2000),
    book_count INT NOT NULL DEFAULT 0,
    rating DOUBLE NOT NULL DEFAULT 0,
    is_public BOOLEAN NOT NULL DEFAULT TRUE,
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);

-- 7. booklist_comment
CREATE TABLE booklist_comment (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    booklist_code VARCHAR(64) NOT NULL,
    user_id BIGINT NOT NULL,
    content VARCHAR(1000) NOT NULL,
    created_at DATETIME NOT NULL
);

-- 8. booklist_follow
CREATE TABLE booklist_follow (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    booklist_code VARCHAR(64) NOT NULL,
    user_id BIGINT NOT NULL,
    UNIQUE KEY uk_booklist_follow (booklist_code, user_id)
);

-- 9. booklist_like
CREATE TABLE booklist_like (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    booklist_code VARCHAR(64) NOT NULL,
    user_id BIGINT NOT NULL,
    UNIQUE KEY uk_booklist_like (booklist_code, user_id)
);

-- 10. activity
CREATE TABLE activity (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    code VARCHAR(64) NOT NULL UNIQUE,
    title VARCHAR(128) NOT NULL,
    description VARCHAR(1000),
    cover_url VARCHAR(255),
    organizer_id BIGINT NOT NULL,
    start_date VARCHAR(64),
    end_date VARCHAR(64),
    location VARCHAR(128),
    participant_count INT NOT NULL DEFAULT 0,
    max_participants INT,
    status VARCHAR(32) NOT NULL DEFAULT 'upcoming',
    approval_status VARCHAR(32) NOT NULL DEFAULT 'pending',
    activity_type VARCHAR(32) DEFAULT 'offline',
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL
);

-- 11. activity_participant
CREATE TABLE activity_participant (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    activity_code VARCHAR(64),
    activity_id BIGINT,
    user_id BIGINT NOT NULL
);

-- 12. private_message
CREATE TABLE private_message (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    sender_id BIGINT NOT NULL,
    receiver_id BIGINT NOT NULL,
    content VARCHAR(2000) NOT NULL,
    is_read BOOLEAN NOT NULL DEFAULT FALSE,
    created_at DATETIME NOT NULL,
    INDEX idx_pm_sender (sender_id),
    INDEX idx_pm_receiver (receiver_id)
);
```
